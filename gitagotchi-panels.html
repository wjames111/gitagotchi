gitagotchi · expanded panel designs

The four main-screen panels, each opened into its full view via its panel hotkey. Same data story throughout: @will, Rust, Zeruko at happiness 82.

vitals · expanded — stat detail (s)

gh pet — stats
┤ vitals · all 10 ├jk select · ↵ open
┤ happiness · how it's built ├

Design intent: the stat panel is the fix-it menu. The selected stat shows its 30-refresh history, the human "why" with concrete facts, its literal API source, and ↵ opens the exact GitHub page that raises it. The right panel makes §3.1 legible: each stat's weighted contribution to happiness, with the misery-cap rule stated where it's relevant instead of buried in docs.

Design intent: rows show real pixel pets, each in its owner's linguist
  color — the list is a crowd, not a table. Selection drives the preview pane instantly from the
  5-minute cache (no network on keystroke). Hibernating friends sink to the bottom with day
  counts — "who's gone quiet" is one glance. The preview's health row says private · by
  design rather than pretending the data doesn't exist.

Design intent: the feed narrates the stat system — every entry names its
  event, its source, and its effect on the pet (+18 hunger),
  which is how users learn the mechanics without a manual. Filter chips map to notification
  reasons. Security entries link out and pre-warn the sick state; messages themselves are never
  shown — the pet is a doorbell, not a mail client.
